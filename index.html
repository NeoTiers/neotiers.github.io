NeoTiers API

REST API for the NeoTiers Minecraft PvP ranking system. All endpoints return JSON. Public routes require no authentication.

Base URL
https://your-backend.vercel.app

Public Routes

GET /api/leaderboard Ranked player list

Returns all players sorted by ELO score. Supports filtering by mode and category, with pagination.

Query Parameters
ParameterTypeDefaultRequiredDescription
mode string comp optional Rating mode — comp or casual
category string all optional Filter by category or all for total ELO
page number 1 optional Page number
limit number 50 optional Results per page (max 100)
Example Request
GET /api/leaderboard?mode=comp&category=sword&page=1
Response 200
{
  "leaderboard": [
    {
      "rank": 1,
      "ign": "Technoblade",
      "minecraft_uuid": "b876ec32-e396-476b-a115-8438d83c67d4",
      "avatar_url": "https://mc-heads.net/avatar/Technoblade/48",
      "ratings": {
        "sword": 1980, "axe": 1750, "smp": 1900,
        "nethpot": 1820, "pot": 1870, "crystal": 1650,
        "mace": 1700, "custom_kit": 1800, "uhc": 1930
      },
      "score": 16400
    }
  ],
  "meta": {
    "mode": "comp", "category": "sword",
    "page": 1, "limit": 50,
    "total_players": 42, "total_pages": 1
  }
}

Webhook

POST /api/match ⚡ webhook secret Submit match result

Called by your Minecraft server plugin after a match ends. Calculates and applies ELO changes to the casual ratings of all involved players.

Requires the header x-neotiers-secret matching MATCH_WEBHOOK_SECRET in your backend .env.
Headers
HeaderRequiredDescription
x-neotiers-secret required Webhook secret from your .env.local
Content-Type required application/json
Body
FieldTypeRequiredDescription
category string required One of the 9 valid categories
match_type string required 1v1, ffa, or team
players array required Array of player result objects
players[].uuid string required Player's Minecraft UUID
players[].won boolean required Whether this player won
players[].placement number optional Final placement (1 = first). Used for FFA ELO.
players[].team string optional Team identifier for team matches
server_id string optional Identifier for your Minecraft server
Example Request Body — 1v1
{
  "server_id": "my-server",
  "category": "sword",
  "match_type": "1v1",
  "players": [
    { "uuid": "b876ec32-e396-476b-a115-8438d83c67d4", "won": true,  "placement": 1 },
    { "uuid": "ec70bcaf-702f-4bb8-b48d-476232a6b527", "won": false, "placement": 2 }
  ]
}
Example Request Body — FFA
{
  "category": "crystal",
  "match_type": "ffa",
  "players": [
    { "uuid": "b876ec32-...", "won": true,  "placement": 1 },
    { "uuid": "ec70bcaf-...", "won": false, "placement": 2 },
    { "uuid": "f2aff236-...", "won": false, "placement": 3 }
  ]
}
Response 201
{
  "match_id": "cb17a983-5ae1-4e6d-83c7-8316d48f2c17",
  "category": "sword",
  "match_type": "1v1",
  "results": [
    { "uuid": "b876ec32-...", "elo_before": 1000, "elo_after": 1016, "delta": 16,  "won": true  },
    { "uuid": "ec70bcaf-...", "elo_before": 1000, "elo_after": 984,  "delta": -16, "won": false }
  ]
}
If a UUID is not found in the database, the player is automatically registered with their UUID as a placeholder IGN. Staff can update the IGN later.

Staff Routes

All staff routes require a valid Supabase session token passed as Authorization: Bearer <token>. The user must exist in the staff table.

POST /api/staff/comp 🔒 staff Set comp ratings

Set one or more comp ELO ratings for a player. Only the fields you include will be updated. All values must be between 0 and 2000.

Body
FieldTypeRequiredDescription
minecraft_uuid string required Player's Minecraft UUID
ratings object required Key/value of category → ELO (0–2000). Partial updates allowed.
reason string optional Reason for the change — stored in tier_history
Example Request Body
{
  "minecraft_uuid": "b876ec32-e396-476b-a115-8438d83c67d4",
  "ratings": {
    "sword": 1980,
    "crystal": 1700
  },
  "reason": "Updated after Bliss tournament"
}
Response 200
{
  "updated": true,
  "player": "Technoblade",
  "changes": { "sword": 1980, "crystal": 1700 }
}
POST /api/staff/players 🔒 staff Add a player
Body
FieldTypeRequiredDescription
ign string required Player's Minecraft IGN
minecraft_uuid string required Player's Minecraft UUID (get from mcuuid.net)
Example Request Body
{
  "ign": "Technoblade",
  "minecraft_uuid": "b876ec32-e396-476b-a115-8438d83c67d4"
}
Response 201
{ "created": true, "ign": "Technoblade" }
DEL /api/staff/players 👑 admin Remove a player
Requires admin role (not just staff). Permanently deletes the player and all their ratings and match history.
Body
FieldTypeRequiredDescription
minecraft_uuid string required UUID of the player to remove
Response 200
{ "deleted": true, "ign": "Technoblade" }

Categories

All 9 valid category values. Use these exact strings in API requests.

sword axe smp nethpot pot crystal mace custom_kit uhc
ELO range: 0 – 2000 per category per mode. Max total ELO is 18,000 (9 × 2000).

ELO System

Casual ELO is auto-calculated from match results. Comp ELO is manually set by staff.

1v1 Matches

Uses standard Elo formula with K=32. Expected score is calculated from the rating difference. Winner gains what loser loses.

FFA Matches

Each player is compared against every other player. Results averaged. Players start at 1000 ELO per category.

Team Matches

Team average ELO is used as the representative rating. All players on the winning team gain ELO, losers lose ELO.

Starting ELO

New players start at 1000 casual ELO in every category. Comp ELO starts at null until set by staff.

Error Codes

StatusMeaningCommon Cause
200 OK Request succeeded
201 Created Resource was created (match posted, player added)
400 Bad Request Missing required fields, invalid category, ELO out of range
401 Unauthorized Missing or invalid webhook secret / staff token
403 Forbidden Token is valid but role is insufficient (e.g. editor trying admin action)
404 Not Found Player UUID not found
500 Server Error Database error or misconfigured environment
Error Response Shape
{ "error": "Human-readable error message" }
